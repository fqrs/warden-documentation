The **Shift Logging** module (shown as **Shifts** in the config menu) lets staff clock in, take breaks, and clock out while giving you a live view of who is on duty.

## How to configure

1. Run **`/config`** or **`w!config`**
2. Select **Shifts**
3. Enable the module and set a **Logging Channel**
4. Use **Configure options...** to manage shift types

## Module settings

| Setting | Description |
| --- | --- |
| **Module State** | Enable or disable the module. |
| **Logging Channel** | Text channel where shift events are logged. |

## Shift types

Use **Configure options...** to manage shift types:

| Action | Description |
| --- | --- |
| **Add Shift Type** | Create a new shift type with custom settings. |
| **Edit Shift Type** | Modify an existing shift type. |
| **Remove Shift Type** | Delete a shift type. |
| **Shift Type List** | View all configured shift types. |

Each shift type supports these fields:

| Field | Description |
| --- | --- |
| **Shift Name** | Display name for the shift type (for example, Moderator). |
| **Quota** | Required hours per period (for example, `10h`, `2d`). |
| **Staff Limit** | Maximum number of staff allowed on duty at once. |
| **Nickname Prefix** | Prefix added to a member's nickname while on duty. |
| **Required Roles** | Roles a member must have to use this shift type. |
| **On Duty Role** | Role assigned while the member is on duty. |
| **Break Role** | Role assigned while the member is on break. |

<Note>
Free servers can configure up to **3** shift types. **Warden+** servers can configure up to **10** shift types.
</Note>
